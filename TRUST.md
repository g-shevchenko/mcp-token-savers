# Trust And Verification

Do not trust this repository because of stars, screenshots, or the author's
identity. Trust it only after you or your agent have inspected the code and run
the verification commands below.

Humanswith.ai MCP Stack is intentionally packaged so Claude Code, Codex, Cursor,
Windsurf, and other agentic IDEs can verify it before installation.

## Quick Agent Check

From a fresh clone:

```bash
git clone https://github.com/g-shevchenko/hwai-mcp-stack.git
cd hwai-mcp-stack
bash scripts/agent-preinstall-check.sh
bash install.sh --dry-run
```

Expected result:

- public release audit passes;
- core doctor reports `services: 6`, `ok: 6`, `needs_attention: 0`;
- dry-run prints every file it would write;
- no `sudo`, `rm -rf`, shell `eval`, token-shaped secrets, or private
  Humanswith.ai references are found in installer paths.

## What The Installer Does

Default install:

- clones or updates this repository under `~/.hwai/hwai-mcp-stack`;
- builds the selected local MCP services with `npm`;
- writes `~/.hwai/mcp-stack/env` as a local placeholder env file;
- updates MCP client config for selected clients;
- writes managed project guidance blocks/rules into the target workspace unless
  `HWAI_MCP_AGENT_DOCS=skip` or `--agent-docs=skip` is set;
- runs `doctor` for the selected profile.

Default install does not:

- use `sudo`;
- install launch agents, daemons, browser extensions, or kernel components;
- require private Humanswith.ai infrastructure;
- require API keys for the default `core` profile;
- send raw code, prompts, screenshots, traces, env files, or lockfiles to a
  hosted service.

Optional external-context services are disabled unless you provide your own
endpoint URLs and bearer keys in `~/.hwai/mcp-stack/env`.

## Machine-Readable Manifest

Agents should read [`trust/hwai-mcp-stack.trust.json`](./trust/hwai-mcp-stack.trust.json)
before install. It declares the expected write paths, profiles, network posture,
forbidden installer patterns, and verification commands.

## Release Verification

Prefer tagged releases over `main` for repeatable installs. A release should
include:

- source archive;
- `SHA256SUMS`;
- SBOM;
- release proof;
- GitHub artifact attestation or SLSA provenance when release artifacts are
  generated by GitHub Actions.

When artifacts are attested, verify provenance with:

```bash
gh attestation verify <artifact> -R g-shevchenko/hwai-mcp-stack
```

Attestation proves where and how an artifact was built. It does not replace
source inspection, dependency review, or local dry-run verification.

## Human-Friendly Verification Policy

The safe answer to "can I install this?" is:

1. inspect `install.sh` and `mcp/install.sh`;
2. run `bash scripts/agent-preinstall-check.sh`;
3. run `bash install.sh --dry-run`;
4. install the `core` profile first;
5. restart the MCP client and run a small local-only test;
6. move to `repo`, `browser-debug`, or `full` only after `core` passes.
